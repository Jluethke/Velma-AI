## Detailed Description of the Drawings

Referring to FIG. 1, a computing system for runtime governance of adaptive learning systems comprises six modules arranged in a pipeline. The domain adapter interface (100) receives divergence measurements from a governed adaptive learning system (118) and provides them to the observation module (102). The observation module (102) receives, from the domain adapter interface (100), a plurality of divergence measurements across a configurable number of measurement axes. The assessment module (104) computes a combined divergence value as a weighted aggregate of the plurality of divergence measurements and computes a trust value by applying an exponential decay function to said combined divergence value. The anticipation module (106) projects the trust value forward over a configurable number of future evaluation steps using divergence trend extrapolation and compares the projected trust value against crisis records stored in a crisis memory bank using a vector similarity function. The modulation module (108) maps the trust value to an authority constraint (116) via linear interpolation between phase-specific authority levels defined by configurable phase thresholds. The enforcement module (110) applies the authority constraint (116) to proposed operational modifications (114) and, when the trust value falls below a configurable rollback threshold, restores both an internal governance state and a domain-specific external state to a prior validated snapshot (122). The audit module (112) produces a governance record at each governance evaluation step and generates a tamper-evident chain (120) by computing a collision-resistant cryptographic hash of each governance record together with the cryptographic hash of the immediately preceding governance record.

Referring to FIG. 2, the trust computation flow comprises two stages within the observation module (102) and the assessment module (104). A plurality of divergence measurement axes, including divergence measurement axis 1 (200), divergence measurement axis 2 (202), divergence measurement axis N (204), and a temporal axis (206), feed into a weighted aggregate (208) to produce a combined divergence value (210). The combined divergence value (210) passes into the exponential decay function (212) which produces a raw trust value (214). The raw trust value (214) passes into the exponential moving average function (216) which produces a smoothed trust value (218). The smoothed trust value (218) passes through an optional post-trust adjustment callback (220) that can only decrease or maintain the trust value, producing the trust value output (222) bounded within a closed interval from zero to one.

Referring to FIG. 3, the anticipation module comprises two parallel subsystems: trust projection and crisis pattern matching. The combined divergence value history (300) feeds into the divergence trend extrapolation (302), which computes a projected divergence value (304). The projected divergence value (304) is used to compute the projected trust value (306) by applying the exponential decay function to the extrapolated divergence value. Separately, the current divergence measurement vector (308) is compared against crisis divergence signatures (313) stored in the crisis memory bank (312) using the vector similarity function (310). When a crisis pattern match (314) is identified, a post-crisis probation period (316) is entered. Both the projected trust value (306) and the post-crisis probation period (316) feed into the preemptive authority constraint (318), which constrains authority when the projected trust value falls within a configurable proximity of the rollback threshold.

Referring to FIG. 4, the modulation module maps the trust value to an authority constraint via linear interpolation between phase-specific authority levels. At trust value zero (400), the system enters the rollback phase (402), where no operational modifications are permitted. Above the configurable rollback threshold, the system enters the frozen phase (404), which prohibits all operational modifications. Above the next configurable phase threshold, the constrained phase (406) permits reduced authority. The phase-specific authority level (408) is computed via linear interpolation within each phase. The nominal phase (410) permits full operational authority at trust value one (412). The scaling factor (414) is bounded within a closed interval from zero to one such that the system can only attenuate or block proposed modifications and cannot amplify them.

Referring to FIG. 5, the two-layer rollback mechanism comprises a snapshot capture (500) that stores two independent layers. The first layer (502) stores snapshots of internal governance state including divergence history (506), trust state (508), and evaluation step count (510). The second layer (504) stores snapshots of domain-specific external state captured via the domain adapter interface state capture (512). Both layers are stored as an immutable deep copy stack (514). When a configurable rollback threshold trigger (516) is met, the rollback execution (518) restores the internal governance state (520) and the domain-specific external state (522) independently to their prior validated conditions.

Referring to FIG. 6, the tamper-evident audit chain comprises a sequence of governance records linked by collision-resistant cryptographic hashes. The genesis hash (600) serves as the initial hash value. Each governance record, including governance record 1 (602), governance record 2 (604), and governance record N (606), includes its data and a cryptographic hash computed from the record contents together with the cryptographic hash of the preceding governance record (624). Each governance record comprises at least a timestamp (610), a domain identifier (612), a combined divergence value (614), a trust value (616), a phase classification (618), an authority constraint (620), and a governance action identifier (622). The session root hash (608) represents the final cryptographic hash of the tamper-evident chain for a complete governance session. Modification of any governance record, such as a modified governance record (626), invalidates the cryptographic hash of that record and all subsequent records (628), thereby rendering any tampering detectable by recomputation of the hash chain.

Referring to FIG. 7, the phase state machine comprises four phases with deterministic transitions. The nominal phase (700) permits full operational authority. When the trust value falls below the configurable phase threshold (708), the system transitions to the constrained phase (702), which permits reduced authority. When the trust value falls below the frozen threshold (710), the system transitions to the frozen phase (704), which prohibits all operational modifications. When the trust value falls below the configurable rollback threshold (712), the system transitions to the rollback phase (706), triggering restoration of a prior validated state. Transitions between phases are deterministic functions of the trust value and the configurable phase thresholds.

Referring to FIG. 8, the domain adapter interface (800) is an abstract interface that enables the governance system to operate identically across different domains. The domain adapter interface (800) comprises a divergence computation method (802) that accepts domain-specific parameters (808) and returns divergence measurements per axis (810) for each configured measurement axis. A state capture method (804) returns a serializable snapshot (812) of domain-specific state. A state restoration method (806) accepts a previously captured snapshot and restores domain-specific state to the restored domain state (814). Multiple concrete implementations, including concrete implementation A (816), concrete implementation B (818), and concrete implementation N (820), substitute different domain-specific logic without modification to the governance system.
