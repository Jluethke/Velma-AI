## Detailed Description of the Drawings

Referring to FIG. 1, a system for runtime governance of an autonomous or semi-autonomous system is shown. The system comprises an expectation interface (100) configured to receive expected system behavior generated by at least one controller or predictive model (120), and an observation interface (102) configured to receive observed system behavior from at least one sensor, estimator, or physical plant (122). A divergence computation module (104) computes a divergence metric between the expected system behavior and the observed system behavior.

The divergence metric is provided to a trust inference module (106), which computes a trust value based at least in part on the divergence metric. The trust inference module (106) includes a neural trust inference component (108), comprising a multi-layer network with a shared encoder and a plurality of output heads. The neural trust inference component (108) receives a feature vector derived from the divergence metric and contextual system state, and produces a trust score and at least one classification signal indicating a degraded or unsafe operating condition, which are used to modulate the trust value.

The trust value is provided to an authority mapping module (110), which computes a permitted control authority as a monotonic non-increasing function of the trust value. The trust value is also provided to a mode management module (112), which selects between a plurality of operating modes based on the trust value.

An enforcement adapter (114) receives the permitted control authority from the authority mapping module (110), the operating mode from the mode management module (112), and a control command from the upstream controller (120). The enforcement adapter (114) is non-bypassable such that all control commands pass through the enforcement adapter (114) prior to actuation, and prevents amplification of any control command. The enforcement adapter (114) produces an enforced control command that is applied to at least one actuator (124).

An audit artifact generator (116) produces, at each time step, a safety state record comprising at least the divergence metric, the trust value, the permitted control authority, the control command, and the enforced control command. The audit artifact generator (116) computes a cryptographic hash of each safety state record and links each safety state record to a cryptographic hash of a prior safety state record to form a tamper-evident audit chain (118).

Referring to FIG. 2, the multi-stage trust evaluation pipeline within the trust inference module (106) is shown. A divergence metric input (200) is processed through a bounded decay function to compute an instantaneous trust value (202). The instantaneous trust value (202) is smoothed by applying exponential moving average smoothing over a configurable temporal window to produce a smoothed trust value (204). The smoothed trust value (204) is modulated with a neural trust score (206) produced by the neural trust inference component (108) to produce a composite trust value (208).

The composite trust value (208) is then processed through a safety envelope check (210), physics threshold checks (212), operational design domain checks (214), and recovery hysteresis with a dwell counter (216). The final trust signal assembly (218) produces a trust value output (220) and operating mode flags indicating degraded or unsafe conditions (222).

Referring to FIG. 3, the neural trust inference component (108) is shown in detail. A feature vector (300) comprising a plurality of elements derived from at least the divergence metric, a measure of observed dynamics, a temporal measure of trust, and at least one contextual signal describing a physical operating condition is provided to a shared encoder (302) implemented as a multi-layer network. The shared encoder (302) produces a latent representation (304) that is fed to a plurality of output heads. A first output head (306) produces a trust score output (312). A second output head (308) produces a classification signal indicating a degraded operating condition (314). A third output head (310) produces a classification signal indicating an unsafe operating condition (316).

Referring to FIG. 4, the mode management module (112) implements a state machine with a plurality of operating modes comprising at least a nominal mode (400), a degraded mode (402), and a failsafe mode (404). Transitions between operating modes are deterministic for identical inputs. In the degraded mode (402), the enforcement adapter (114) scales the control command by a configurable authority factor. In the failsafe mode (404), the enforcement adapter (114) replaces the control command with a predetermined failsafe command. Recovery from the failsafe mode (404) to the nominal mode (400) requires a configurable dwell period (406) during which the trust value must remain above a recovery threshold, with the rate of recovery modulated by the current actuator authority level.

Referring to FIG. 5, the authority mapping module (110) maps the trust value to a permitted control authority using a monotonic non-increasing function (504). As the trust value decreases from the nominal zone (510) through the degraded zone (508) to the failsafe zone (506), the permitted control authority decreases correspondingly. The degradation threshold (514) marks the boundary between the nominal zone and the degraded zone. The failsafe threshold (512) marks the boundary between the degraded zone and the failsafe zone. Within each zone, the authority value is computed by linear interpolation.

Referring to FIG. 6, the enforcement adapter (114) is shown receiving a control command from an upstream controller (600). The enforcement adapter (602) is non-bypassable and applies constraints selected from a hierarchy of constraint severity levels. The hierarchy comprises at least an advisory level (604) producing no modification, a bounded scaling level (606), a strict clamping level (608), and a failsafe override level (610). The constraint of highest severity takes precedence when multiple constraints are applicable. An anti-amplification invariant check (612) ensures the enforcement adapter prevents amplification of any control command. The enforced control command (614) is output to the actuator, and a safety state record (616) is provided to the audit artifact generator (116).

Referring to FIG. 7, the envelope recovery state machine implements a formal safety envelope defining configurable bounds on allowable system behavior. The state machine transitions between an inactive state (700), an unsafe state (702), and a recovering state (704). Violation of the safety envelope forces the permitted control authority to a failsafe value in the unsafe state (702) regardless of the trust value. Recovery from the unsafe state (702) requires a configurable number of consecutive time steps (706) with no envelope violation before returning to the inactive state (700).

Referring to FIG. 8, the operational design domain boundary module (800) determines whether operation is within a declared operating domain. The operating domain is defined by configurable bounds on at least one of: system velocity (802), environmental conditions (804), sensor agreement (806), and operational parameters (808). An ODD boundary evaluation (810) determines whether all bounds are satisfied. If within bounds (812), normal operation continues. If any bound is violated, an ODD violation is detected (814), the trust value is forced to unsafe (816), and a mode transition is triggered (818).

Referring to FIG. 9, the tamper-evident audit chain is shown across multiple time steps. Each safety state record (900, 902, 904) comprises the divergence metric (912), the trust value (914), the permitted control authority (916), and the control command with the enforced control command (918). A cryptographic hash (906) is computed for each safety state record. Each safety state record is linked to the cryptographic hash of the prior safety state record (908) to form the tamper-evident audit chain. A root hash (910) represents the terminal cryptographic hash of the audit chain for a runtime session. Modification of any safety state record alters all subsequent cryptographic hashes in the chain, enabling post-event reconstruction of all governance decisions for certification, regulatory review, or forensic analysis.
