## INDEPENDENT CLAIMS

### Claim 1 -- System

**1.** A system for runtime governance of an autonomous or semi-autonomous system, comprising:

   a) an expectation interface configured to receive expected system behavior generated by at least one controller, planner, or predictive model;

   b) an observation interface configured to receive observed system behavior from at least one sensor, estimator, or physical plant;

   c) a divergence computation module configured to compute a divergence metric between the expected system behavior and the observed system behavior;

   d) a trust inference module configured to compute a trust value based at least in part on the divergence metric, the trust value being bounded within a predefined range;

   e) a neural trust inference component within the trust inference module, the neural trust inference component comprising a multi-layer network with a shared encoder and a plurality of output heads, each output head producing a distinct governance signal, wherein at least one output head produces a trust score and at least one additional output head produces a classification signal indicating a degraded or unsafe operating condition;

   f) an authority mapping module configured to compute a permitted control authority as a monotonic non-increasing function of the trust value;

   g) an enforcement adapter configured to apply the permitted control authority to at least one control command from an upstream controller before the control command is applied to an actuator, wherein the enforcement adapter is non-bypassable such that all control commands pass through the enforcement adapter prior to actuation, and wherein the enforcement adapter prevents amplification of any control command;

   h) an audit artifact generator configured to produce, at each time step, a safety state record comprising at least the divergence metric, the trust value, the permitted control authority, the control command, and an enforced control command, and to compute a cryptographic hash of each safety state record and to link each safety state record to a cryptographic hash of a prior safety state record to form a tamper-evident audit chain;

   wherein the system operates independently of internal logic, parameters, or structure of the controller or predictive model, and wherein the enforcement adapter prevents actuation that exceeds the permitted control authority when the trust value indicates unsafe operation.

### Claim 2 -- Method

**2.** A method for runtime governance of an autonomous or semi-autonomous system, comprising:

   a) receiving expected system behavior from a controller or predictive model;

   b) receiving observed system behavior from at least one sensor or estimator;

   c) computing a divergence metric between the expected system behavior and the observed system behavior;

   d) computing an instantaneous trust value from the divergence metric using a bounded decay function;

   e) computing a smoothed trust value by applying temporal smoothing to the instantaneous trust value over a configurable window;

   f) computing a neural trust score by processing a feature vector derived from the divergence metric and contextual system state through a multi-layer network having a plurality of output heads;

   g) computing a composite trust value by combining the smoothed trust value and the neural trust score;

   h) mapping the composite trust value to a permitted control authority using a monotonic non-increasing function;

   i) enforcing the permitted control authority on at least one control command prior to actuation through a non-bypassable enforcement path that prevents amplification of said control command, thereby producing an enforced control command;

   j) generating a cryptographic hash of a safety state comprising at least the divergence metric, the composite trust value, the permitted control authority, and the enforced control command; and

   k) linking the cryptographic hash to a hash of a prior safety state record to form a tamper-evident audit chain;

   wherein the method operates independently of internal structure of the controller or predictive model.

### Claim 3 -- Computer-Readable Medium

**3.** A non-transitory computer-readable medium storing instructions that, when executed by at least one processor, cause the at least one processor to perform operations comprising:

   a) computing a divergence metric between expected system behavior received from a controller and observed system behavior received from at least one sensor;

   b) computing a trust value from the divergence metric using a bounded decay function and temporal smoothing;

   c) processing a feature vector through a neural trust inference component having a shared encoder and a plurality of output heads to produce a trust score and at least one classification signal indicating an operating condition;

   d) computing a permitted control authority as a monotonic non-increasing function of the trust value;

   e) enforcing the permitted control authority on at least one control command through a non-bypassable enforcement path that prevents amplification;

   f) generating a tamper-evident audit chain by cryptographically hashing each safety state record and linking each hash to a hash of a prior safety state record; and

   g) selecting an operating mode from a plurality of operating modes based on the trust value, and transitioning between operating modes deterministically.

## DEPENDENT CLAIMS

### Trust Computation Claims

**4.** The system of claim 1, wherein the trust inference module computes the trust value through a multi-stage pipeline comprising: computing an instantaneous trust value from the divergence metric using a bounded decay function; computing a smoothed trust value by applying exponential moving average smoothing over a configurable temporal window; and modulating the smoothed trust value with a neural trust score produced by the neural trust inference component.

**5.** The system of claim 4, wherein the bounded decay function produces a trust value that decreases exponentially as the divergence metric increases, with a configurable decay rate.

**6.** The system of claim 1, wherein the trust inference module is configured to reduce the trust value when the divergence metric persists above a configurable threshold over multiple consecutive time steps.

**7.** The system of claim 1, wherein the neural trust inference component receives a feature vector comprising a plurality of elements derived from at least: the divergence metric, a measure of observed dynamics, a temporal measure of trust, and at least one contextual signal describing a physical operating condition.

### Authority and Mode Management Claims

**8.** The system of claim 1, further comprising a mode management module configured to select between a plurality of operating modes based on the trust value, the plurality of operating modes comprising at least a nominal mode, a degraded mode, and a failsafe mode, wherein transitions between operating modes are deterministic for identical inputs.

**9.** The system of claim 8, wherein in the degraded mode, the enforcement adapter scales the control command by a configurable authority factor, and in the failsafe mode, the enforcement adapter replaces the control command with a predetermined failsafe command.

**10.** The system of claim 1, wherein the enforcement adapter applies a constraint selected from a hierarchy of constraint severity levels, the hierarchy comprising at least an advisory level producing no modification, a bounded scaling level, a strict clamping level, and a failsafe override level, wherein the constraint of highest severity takes precedence when multiple constraints are applicable.

### Enforcement Adapter Claims

**11.** The system of claim 1, wherein the enforcement adapter is configured to receive declarative constraints specifying severity, maximum scaling factors, and maximum allowed values, and to enforce said constraints without interpreting the trust value or governance logic that produced them.

**12.** The system of claim 11, wherein the enforcement adapter applies constraints to control commands generated by any upstream controller without requiring modification of the controller's internal logic, state, or execution semantics.

### Safety Envelope Claims

**13.** The system of claim 1, further comprising a formal safety envelope defining configurable bounds on allowable system behavior, wherein the safety envelope is declared independently of any learned model, is immutable during runtime operation, and is enforced regardless of the trust value.

**14.** The system of claim 13, wherein violation of the safety envelope forces the permitted control authority to a failsafe value regardless of the trust value.

**15.** The system of claim 13, wherein the safety envelope parameters are cryptographically bound to the tamper-evident audit chain.

**16.** The system of claim 13, further comprising an envelope recovery state machine configured to transition between an inactive state, an unsafe state, and a recovering state, wherein recovery from the unsafe state requires a configurable number of consecutive time steps with no envelope violation before returning to the inactive state.

### Recovery Hysteresis Claims

**17.** The system of claim 8, wherein recovery from the failsafe mode to the nominal mode requires a configurable dwell period during which the trust value must remain above a recovery threshold, and wherein a rate of recovery is modulated by a current actuator authority level.

### Operational Design Domain Claims

**18.** The system of claim 1, further comprising an operational design domain boundary module configured to determine whether operation is within a declared operating domain, the operating domain being defined by configurable bounds on at least one of: system velocity, environmental conditions, sensor agreement, and operational parameters.

**19.** The system of claim 18, wherein violation of the declared operating domain forces the trust value to an unsafe value and triggers a mode transition.

### Audit and Evidence Claims

**20.** The system of claim 1, wherein the audit artifact generator produces a root hash representing a terminal cryptographic hash of the tamper-evident audit chain for a runtime session.

**21.** The system of claim 1, wherein modification of any safety state record in the tamper-evident audit chain alters all subsequent cryptographic hashes in the tamper-evident audit chain.

**22.** The system of claim 1, wherein the safety state records in the tamper-evident audit chain enable post-event reconstruction of all governance decisions for certification, regulatory review, or forensic analysis.

### System Properties Claims

**23.** The system of claim 1, wherein the system operates deterministically such that identical inputs and state produce identical trust values, authority constraints, and enforcement decisions.

**24.** The system of claim 1, wherein the system does not perform online learning in a governance path of the system.

**25.** The system of claim 1, wherein the system does not generate autonomous control commands.
